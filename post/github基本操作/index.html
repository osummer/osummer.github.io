github使用指南

1. 新建账户与仓库

在github.com中 osummer 账户下新建 easyANOVA 仓库

2. ssh公钥建立

首先在本地创建ssh key，在cmd中执行创建ssh key

$ ssh-keygen -t rsa -C "your_email@youremail.com"

your_email@youremail.com改为你在github上注册的邮箱，之后会要求确认路径和输入密码，我们这使用默认的一路回车就行。成功的话会在~/下生成.ssh文件夹，进去，打开id_rsa.pub，复制里面的key。

3. ssh公钥上传

回到github上，进入 Account Settings（账户配置），左边选择SSH Keys，Add SSH Key,title随便填，粘贴在你电脑上生成的key。
验证是否成功：

$ ssh -T git@github.com

4. 本地仓库初始化与提交

设置信息

$ git config --global user.name "your name"
$ git config --global user.email "your_email@youremail.com"

进入要上传的仓库，右键git bash，添加远程地址：

$ git remote add origin git@github.com:yourName/yourRepo.git

或者重新创建本地仓库：

echo "# easyANOVA" >> README.md
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/osummer/easyANOVA.git
git push -u origin main

5. 检出仓库

# 本地仓库
$ git clone /path/to/repository 
# 如果是远程仓库，则为：
$ git clone username@host:/path/to/repository

6. git工作流

你的本地仓库由 git 维护的三棵"树"组成。第一个是你的 工作目录，它持有实际文件；第二个是 暂存区（Index），它像个缓存区域，临时保存你的改动；最后是 HEAD，它指向你最后一次提交的结果。

（1）提出更改 (git add)

提出更改是把待提交内容添加到暂存区，代码为：

git add <filename> #单个文件
git add * #或者所有内容

（2）提交到head (git commit)

git commit -m "代码提交信息"

实际提交了改动，但还未到远程仓库。

（3）提交到远程（git push）

现在，改动现在已经在本地仓库的 HEAD 中了。将这些改动提交到远端仓库：


git push origin master

7. FAQ

出现ssh登录错误，比如：

$ ssh -T git@github.com
kex_exchange_identification: read: Connection reset by peer
Connection reset by 20.205.243.166 port 22

解决方法：

git remote set-url origin HTTPS-url
#https-url为仓库的地址“https://github.com/osummer/easyANOVA.git”
